

# Fix Inconsistent Signature Blocks and Grey Footer in Complaint Letters

## Problem Summary

When letters are generated or regenerated, three issues occur inconsistently:

1. **Duplicate signature blocks** -- "Yours sincerely" appears twice, each followed by name, role, practice, and address details
2. **Phone and email in large blue text** -- contact lines containing bold markers (`**Phone**: ...`) are incorrectly rendered as signatory names (large, blue, bold)
3. **Grey footer showing duplicate practice details** -- the grey section at the bottom repeats the practice name, address, and sometimes shows "*Letterhead/Logo Here*" placeholder text. Only the "This letter was generated by Notewell AI..." line should remain

## Root Causes

There are issues in both the **AI prompts** (edge functions) and the **front-end rendering** (FormattedLetterContent component):

| Cause | Location | Detail |
|-------|----------|--------|
| Acknowledgement prompt tells AI to put practice details at the top AND in the signature | `generate-complaint-acknowledgement/index.ts` (lines 238-253, 297-321) | AI sometimes produces a practice header block + a full signature block, creating duplication |
| Outcome letter code appends practice address after the signature | `generate-complaint-outcome-letter/index.ts` (lines 528-531) | `outcomeLetter += practiceDetails.address` adds a raw address block after the AI's signature, contradicting the prompt's own rule of "do NOT repeat the address" |
| Grey footer renders `headerLines` | `FormattedLetterContent.tsx` (lines 377-391) | Lines captured before the date (practice name/address) are re-rendered in a grey box at the bottom |
| Signature parser treats bold phone/email as names | `FormattedLetterContent.tsx` (line 350) | `if (trimmedLine.includes('*') \|\| index === 1)` matches any line with asterisks, including `**Phone**: 01234...`, rendering it as large blue bold text |
| No deduplication of "Yours sincerely" | `FormattedLetterContent.tsx` (lines 207-214) | Every line matching "Yours sincerely" starts a new signature section without checking if one already exists |

## Changes

### 1. `src/components/FormattedLetterContent.tsx` -- Fix rendering

**Grey footer** (lines 377-399):
- Remove the `headerLines` rendering block entirely from the footer
- Keep only the "This letter was generated by the Notewell AI Complaints Management System" line
- The grey background section becomes just the single attribution line

**Signature deduplication** (lines 207-214):
- Only the FIRST "Yours sincerely" / "Yours faithfully" / "Kind regards" triggers the signature section
- If a second one is encountered, skip it and all subsequent lines until a clearly non-signature line appears (prevents double blocks)

**Phone/email detection in signature** (lines 349-370):
- Add detection for lines starting with `Phone`, `Email`, `Tel`, `**Phone**`, `**Email**` etc.
- Render these as normal grey text (same style as address lines), not as large blue bold name text
- The `index === 1` fallback that currently forces the second signature line to be treated as a name will be tightened: only treat it as the signatory name if it does NOT contain phone/email/address patterns

**Filter placeholder text**:
- Add `*Letterhead/Logo Here*` and `[Letterhead]` to the content cleaning regex so they are stripped before rendering

### 2. `supabase/functions/generate-complaint-acknowledgement/index.ts` -- Tighten prompt

**System prompt** (lines 238-255):
- Remove "Start with the practice logo/letterhead" instruction -- the logo is handled by the front-end component via the embedded HTML comment
- Clarify: "Do NOT include any practice header or letterhead text at the start of the letter. Start directly with the date."
- Clarify: "Include EXACTLY ONE signature block. Do not repeat the signatory name, practice name, or address anywhere else in the letter."

**User prompt** (lines 295-321):
- Remove instructions to format phone/email with bold labels (`**Phone**:`, `**Email**:`)
- Instead: "Include the practice phone and email as plain text lines within the body of the letter where you mention how to contact the practice. Do NOT include them in the signature block."
- This prevents the rendering issue where bold markers cause phone/email to display as names

### 3. `supabase/functions/generate-complaint-outcome-letter/index.ts` -- Remove trailing address

**Remove appended address** (lines 528-531):
- Delete the block that appends `practiceDetails.address` after the letter content
- The prompt at line 490 already says "DO NOT include the practice address in the signature block -- it should only appear ONCE in the letter header"
- This code contradicts that rule and causes duplicate address blocks

**Strengthen signature instruction** (line 488-493):
- Add: "The letter must contain EXACTLY ONE 'Yours sincerely' signature block. Do not repeat the signatory name, practice details, or address after the signature."

### 4. `supabase/functions/regenerate-outcome-letter/index.ts` -- Add signature dedup rule

**System prompt** (lines 38-62):
- Add instruction: "Ensure the revised letter contains exactly ONE signature block ending with 'Yours sincerely'. Remove any duplicate signature sections, repeated practice details, or trailing address blocks."
- Add: "Do not include '*Letterhead/Logo Here*' or similar placeholder text."
