

## Problem

The email Word attachment is generated by a simplistic markdown-to-HTML converter inside the `generate-policy` edge function (lines 876–948). It produces a basic HTML file saved as `.doc` with:

- **Oversized logo** — set to `max-width:90px;max-height:90px` but Word ignores CSS `max-*` properties, rendering the image at full native resolution
- **Poor table formatting** — tables lack proper borders, header shading, cell padding
- **No document control table** — the professional version/author/dates block is missing
- **Inconsistent fonts/spacing** — no page margins, line spacing, or heading hierarchy matching the web version
- **No footer** — no practice details or page numbers

Meanwhile, the client-side `generatePolicyDocx.ts` produces a polished Word document using the `docx` library with proper heading styles, NHS colour scheme, document control table, sized logo, footers, and well-formatted tables.

## Solution

Rewrite the Word attachment HTML generation in the `finalise` step of `supabase/functions/generate-policy/index.ts` to produce Word-compatible HTML that closely matches the quality of the client-side export.

### Changes (single file: `supabase/functions/generate-policy/index.ts`)

1. **Fix logo sizing** — Use explicit `width` and `height` attributes (not CSS `max-*`) capped at 80×60px. Word respects HTML attributes, not CSS max constraints.

2. **Add document control table** — Insert the same Version / Effective Date / Review Date / Author / Approved By table that appears in the web download, using Word-compatible table styling.

3. **Improve `mdToHtml` converter**:
   - Proper heading hierarchy with NHS blue colours and appropriate font sizes
   - Table header rows get shaded background (`#EFF6FF`) and bold text
   - Separator rows (`---|---`) are stripped cleanly
   - Bullet points use proper `<ul>` nesting with correct margins
   - Numbered section headings (e.g. `1. PURPOSE`) rendered as styled headings, not list items
   - Sub-numbered headings (e.g. `1.1`, `2.3`) get sub-heading styling

4. **Add Word page setup** — Use `@page` margins (2.54cm) and `mso-` prefixed styles for proper Word rendering of fonts, line spacing, and paragraph spacing.

5. **Add footer** — Practice name + "Generated by Notewell AI" + date centred at the bottom.

6. **Fix file extension** — Keep `.doc` (HTML-based Word) since we cannot use the `docx` library in Deno, but the formatting quality will now be substantially better.

### What stays the same
- The client-side `generatePolicyDocx.ts` (web download) is untouched — it already works well
- The email sending logic, attachment encoding, and base64 conversion remain identical
- No database changes needed
